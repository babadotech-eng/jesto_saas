TECH STACK
- Frontend: React + Vite + TailwindCSS

- Routing: React Router v6
- State: React Context + hooks

IMPORTANT SUPABASE SETUP

- Use supabase-js v2 client.
- All tables must use Row Level Security (RLS) with auth.uid() = user_id policies.
- Create profiles and subscriptions automatically via a trigger on auth.users insert.
- Use these exact table names in Portuguese:
  - perfis (id UUID references auth.users)
  - produtos (id, user_id, nome, categoria, preco_venda, custo_mao_obra, frete, imposto_pct, taxa_cartao_pct, taxa_app_pct, comissao_pct)

  - despesas_fixas (id, user_id, descricao, valor, categoria)
  - lancamentos (id, user_id, descricao, tipo [receita|despesa], valor, data)
  - assinaturas (id, user_id, plano, status, valido_ate)

Generate and run the full SQL schema with:

FINANCIAL CALCULATIONS
Auto-calculate on every product form change:
- CMV = SUM(quantidade × preco_unitario × fator_correcao)
- Custo variável total = CMV + custo_mao_obra + frete
- Imposto = preco_venda × imposto_pct / 100
- Taxa cartão = preco_venda × taxa_cartao_pct / 100
- Taxa app = preco_venda × taxa_app_pct / 100
- Comissão = preco_venda × comissao_pct / 100
- Margem de contribuição = preco_venda − CMV − impostos − taxas − comissão
- Margem de contribuição % = margem / preco_venda × 100
- Ponto de equilíbrio contábil = despesas_fixas / margem_media
- Ponto de equilíbrio econômico = (despesas_fixas + lucro_desejado) / margem_media

Show all values with 2 decimal places.
Show margin percentage with color: green if > 30%, yellow if 15–30%, red if < 15%.
