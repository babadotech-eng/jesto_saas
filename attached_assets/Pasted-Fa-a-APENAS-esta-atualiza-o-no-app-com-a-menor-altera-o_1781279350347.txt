OBJETIVO
Criar um **Excel modelo de Ficha Técnica** para **download** e posterior **upload** na aba **Ficha Técnica**, de forma que:
- seja facilmente entendido pelo usuário leigo
- seja facilmente preenchido
- seja lido corretamente pelo app
- seja compatível com os **produtos já cadastrados/lidos pelo app**

ESCOPO EXATO
1. Trabalhar somente na funcionalidade da aba **Ficha Técnica**.
2. Adicionar uma forma de **baixar um modelo Excel de ficha técnica**.
3. Garantir que esse arquivo possa depois ser **reenviado/uploadado** na própria aba Ficha Técnica e ser interpretado corretamente pelo app.
4. O modelo deve ser inteligente e amigável para o usuário final, sem comprometer a estrutura necessária para importação.

OBJETIVO FUNCIONAL DO EXCEL
Esse Excel deve servir como ponte entre:
- os **produtos existentes do app**
- os **insumos**
- a composição da **ficha técnica**

Ou seja:
- cada linha do Excel deve representar um item da ficha técnica
- o app deve conseguir entender a qual **produto** aquela linha pertence
- o app deve conseguir entender qual **insumo** está sendo usado
- o app deve conseguir entender a **quantidade** e a **unidade** usada na receita/composição

ESTRUTURA RECOMENDADA DO ARQUIVO
Criar um arquivo `.xlsx` com estrutura simples, robusta e amigável.

Sugestão de abas:
1. **Instrucoes**
2. **FichaTecnica**
3. **Exemplo** (opcional, mas recomendado se ajudar o usuário)

DETALHAMENTO DAS ABAS

ABA 1 — Instrucoes
- explicar em linguagem simples como preencher
- explicar que não deve alterar os nomes das colunas da aba principal
- explicar que cada linha representa 1 insumo usado em 1 produto
- explicar que, se um produto usa 5 insumos, ele terá 5 linhas
- explicar o formato esperado dos campos
- explicar quais colunas são obrigatórias
- explicar como salvar e subir o arquivo no app

ABA 2 — FichaTecnica
Essa deve ser a aba principal de importação.
Criar colunas claras e estáveis, com nomes simples e legíveis, mas compatíveis com a leitura do app.

Estrutura sugerida:
- `produto_nome`
- `produto_categoria` (se fizer sentido para ajudar consistência)
- `insumo_nome`
- `quantidade`
- `unidade`
- `fator_correcao` (opcional, se a lógica do app usar isso)
- `observacao` (opcional, apenas se o importador tolerar isso sem quebrar)
- `ordem_preparo` (opcional, apenas se útil e seguro)

IMPORTANTE:
- se o app já tiver um formato interno esperado para ficha técnica, seguir o formato já existente do sistema
- se já houver parser/importador pronto, adaptar o Excel ao parser existente em vez de refatorar a lógica toda
- priorizar compatibilidade real com a leitura atual do app

ABA 3 — Exemplo
- incluir algumas linhas de exemplo realistas
- usar produtos já existentes no app

- mostrar claramente como repetir o produto em múltiplas linhas para compor a ficha técnica
